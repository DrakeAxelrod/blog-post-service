Detailed solution

If you've been following along then you'll have your .opvn file (for openvpn) so that you can ssh into the machine. If you don't want to use your local machine you can always use HackTheBox's browser based machine.

  • If you haven't been following along you can checkout the previous post HackTheBox - Meow
  • you can also check out HackTheBox Access Guide

Now that your hopefully connected to the machines we can get going.

  1. First things first lets run nmap
sudo nmap TARGET_IP # replace TARGET_IP with the ip address of your target machine

  1. run nmap with the -sV flags
sudo nmap -sV TARGET_IP # replace TARGET_IP with the ip address of your target machine

  1. try to connect
ftp TARGET_IP # replace TARGET_IP with the ip address of your target machine

  1. at this point we get prompted to provide a username and password. But after checking out the documentation on HackTheBox and some google foo I learned that when misconfigured a FTP service allows for anonymous accounts to access the service.

  1. This should get you in and provide you with a new prompt if you then run ls you'll see the flag.txt

  1. finally lets get the flag onto our machine so we can read it.

  1. finally on our local machine we just cat the flag.txt
cat flag.txt # make sure your in the directory in which you the flag.txt got downloaded to.

Question and Answer Section

  1. What does the 3-letter acronym FTP stand for?
  • File Transfer Protocol
  1. What communication model does FTP use, architecturally speaking?
  • Client-Server model
  1. What is the name of one popular GUI FTP program?
  • FileZilla
  1. Which port is the FTP service active on usually?
  • 21 TCP
  1. What acronym is used for the secure version of FTP?
  • SFTP
  1. What is the command we can use to test our connection to the target?
  • ping
  1. From your scans, what version is FTP running on the target?
  • vsftpd 3.0.3
  1. From your scans, what OS type is running on the target?
  • Unix
